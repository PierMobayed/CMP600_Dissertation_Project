📘 DEVELOPER IMPLEMENTATION & DOCUMENTATION PLAN
Project:Integrated MultiApplication Logistics Big Data Visualisation System
Purpose:Develop a fully working prototype system (Backend + Dashboard + Client App + Driver App) that aligns with an academic dissertation (CMP600) and supports live demonstration during Viva Voce.
1. General Instructions for the Developer
This is an academic project, not a commercial system.
System MUST be fully working, even if simplified.
Focus on: 
functionality
clarity
stability
Simulated data only (no real users, no personal data).
Code decisions must be: 
simple
explainable
demonstrable live.

3. Development Phases & Deliverables
PHASE 1 – Backend (CORE PRIORITY)
Location:Source_Code/backend/
Goal:Create a clouddeployed backend that connects all applications.
MUST implement:
REST API
SQL database (SQLite/PostgreSQL)
Simulated GPS & shipment events
Required Endpoints (minimum):
GET /dashboard/overview
GET /dashboard/map
GET /shipments/{id}/tracking
POST /drivers/{id}/location
POST /simulation/gps/start
POST /simulation/status/start
✅ Backend must be deployable to cloud (Render / Railway / Azure).
